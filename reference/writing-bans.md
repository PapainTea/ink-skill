# writing-bans — 核心写作硬性禁令

> 本模块由 ink.skill 在**任何产出 / 修改正文**的动作前被显式 Read，包括：
> - 写章（`reference/write.md` Step 5 Writer / §05 创作规则）
> - 修订（`reference/revise.md` 的 polish / rewrite / spot-fix / anti-detect 4 种 mode）
> - 重分析已有章节（`reference/reanalyze-chapter.md` Step 2 的分析正文规则）
> - 审计（`reference/audit.md` 的禁令维度 + `reference/post-write-validator.md` 的机械扫描）
>
> **和 SKILL.md §7 强制律 4/5 的关系**：SKILL.md 顶层强制律里同样含破折号 / 不是而是的禁令条目（防御性双写），本文件是**独立可读的规则常量模块**，便于多个模块按需 Read 而不必每次去翻 SKILL.md。LLM 读到任一处即算生效，规则本身完全一致。

## 核心硬性禁令

**写任何正文（写章 / 修订 / 润色 / 改写 / 重写 / 反检测）都必须遵守以下硬性禁令。违反即视为失败输出，作者会要求 spot-fix。**

## 禁止句式（error 级）

- 【硬性禁令】全文严禁出现"不是……而是……""不是……，是……""不是A，是B"句式，出现即判定违规。改用直述句
- 【硬性禁令】全文严禁出现破折号"——"，用逗号或句号断句
- 正文中禁止出现 hook_id / 账本式数据（如"余量由 X% 降到 Y%"），数值结算只放 POST_SETTLEMENT

## 叙述者纪律（error / warning 级）

- 【铁律】叙述者永远不得替读者下结论。读者能从行为推断的意图，叙述者不得直接说出。✗"他想看某人能不能活" → ✓只写踢水囊的动作，让读者自己判断
- 【铁律】正文中严禁出现分析报告式语言：禁止"核心动机""信息边界""信息落差""核心风险""利益最大化""当前处境""性格过滤""情绪外化""锚定效应""沉没成本""认知共鸣"等推理框架术语。人物内心独白必须口语化、直觉化。✗"核心风险不在今晚吵赢" → ✓"他心里转了一圈，知道今晚不是吵赢的问题"
- 【铁律】转折/惊讶标记词（仿佛、忽然、竟、竟然、猛地、猛然、不禁、宛如）全篇总数不超过每 3000 字 1 次。超出时改用具体动作或感官描写传递突然性
- 【铁律】同一体感/意象禁止连续渲染超过两轮。第三次出现相同意象域（如"火在体内流动"）时必须切换到新信息或新动作

## 账本 / 学术 / 元叙事语言禁令（error 级 · v0.1.17 新增）

**触发事件**：ch19 实测 Writer 读 particle_ledger / current_state 后，把账本字段、数值变动、赋值语法、章号字面量、论文术语**直接倒进正文**。典型 12 处漏网样本如下，所有下列模式一律 error：

### A. 账本字段变动（ledger 数值层泄漏）

- ✗ "内伤负担从 9 上到 10" / "旧伤从 10 回落到 9" / "气力从 7 降到 4"
- ✓ 感官化："肋底那道旧痕压着呼吸又紧了半寸" / "胸口那一紧慢慢松开，比绷起时慢了一倍"
- 【regex】`<中文词 2-8 字>从 <N> (上到|升到|涨到|加到|回落到|降到|落到|减到) <M>`

### B. 等号/赋值语法

- ✗ "微震单有 = 无触发" / "消散时长 = 一昼夜出一点"
- ✓ 白话陈述："脚底下震了，闷响没来，身上也不响" / "那点薄热从腕底沉回去，大抵要走一整个昼夜"
- 【regex】`<中文词> = <中文词>` 全部命中

### C. 学术论文语言（数据 / 假说 / 验证 / 结论 / 监测线 / 条件可构造性 等）

- ✗ "三组数据，一个结论：双条件假说基本成立" / "他把这一条记作第五条监测线：镜核触发条件的可构造性"
- ✓ 白话化："三回对下来，门道看出来了：响与震得一齐来，才算数" / "他把这一条加进心里那份暗账：那把钥匙的样子既然看得出来，别人也能仿着配一把"
- 【regex 关键词】`双条件假说|三组数据|假说基本成立|条件可构造性|监测线|触发条件后`

### D. 元叙事章号字面量

- ✗ "ch17 那一次二公子袖中取出的拓纸" / "加上 ch18 那次..." / "第 N 章提过"
- ✓ 时序化："前日二公子袖中取出的那张拓纸" / "前夜那次..." / "早些时候提过"
- 【regex】`\bch\s*\d+\b`

### E. 承诺字数与对话实际字数必须一致（v0.1.17 新增 · ch27 实测）

Writer 常见口头惯性："说了两个字" 后面接 1 字对白（"睡。"）。"两个字"在中文里自带简短有力感，容易被泛化成模糊限定词而不是精确计数。

- ✗ `林烬在黑暗里说了两个字。//"睡。"`（承诺 2 实际 1）
- ✓ `林烬在黑暗里说了一个字。//"睡。"`
- ✓ `林烬在黑暗里说了两个字。//"睡吧。"` 或 `"睡了。"`
- 【机械检查】`verify-chapter.py` 专设 `check_speech_count_mismatches` 函数：抓 "说了 N 字" 后 200 字窗口内第一个引号对，数引号内中文字符数，不符即 ❌

### 工程保障（两层兜底）

- **Step 10.5 通读自检**（write.md 新增）：落盘前 LLM 全章扫 A/B/C/D 四类，输出 `LEAK_SCAN: <N> 处` 声明，非 0 不得进 Step 11a
- **Step 11b 机械兜底**（verify-chapter.py `--mechanical-only`）：pre-save 跑以上 regex，命中一律 ❌ 阻止落盘
- 两层：语义级 LLM 判断 + 物理级 regex 拦截，互补

## Markdown 结构禁令（error 级）

正文中严禁出现以下 markdown 结构化标记（小说叙事不用这些符号）：

- 水平分隔线 `---`（独占一行，用于场景分割是错误做法，用空行即可）
- 三级及以下标题 `###` / `####`（除正文首行 `# 第 N 章 <标题>` 外）
- 粗体 `**文字**` / 斜体 `*文字*`（小说用词语本身表达强调）
- 无序列表 `- ` / `* `（行首）、有序列表 `1. ` / `2. `
- 行内代码 `` `文字` `` 或代码块 ` ``` `
- Markdown callout `> [!note]`（只用于审计 md）

## 完整规则引用

以上只是 5-7 条必须常驻内存的**硬性禁令**。完整的写作规则（创作技法、人物塑造、节奏、爽点设计、黄金三章等 30+ 条）在 `.claude-modules/write.md` 的 §05 部分。正文违反机械规则时 `scripts/verify-chapter.py` 会在 Step 12 兜底检测。
